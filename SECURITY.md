# Security Policy

Dossier is a personal project, not a commercially supported product — there's
no dedicated security team and no SLA, but reports are taken seriously and
looked at promptly.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for a suspected security
vulnerability. Instead, use GitHub's private reporting flow:

1. Go to the [Security tab](../../security) of this repository.
2. Click **"Report a vulnerability"**.

This opens a private advisory visible only to the maintainer, so the issue
isn't disclosed publicly before a fix is available.

## Scope

This is a self-hosted, single-user tool (run locally or via the included
Docker Compose setup) — there is no hosted/multi-tenant deployment to report
issues against. Reports about the application code, the Dockerfile/container
setup, or the CI/CD configuration are all in scope.
